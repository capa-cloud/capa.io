Capa: Cloud Application API

Write Once, Run Anywhere
Mecha Architecture-based Multi-Runtime SDK for Java applications

Overview Get Started GitHub

Why Capa?

Capa (Cloud Application API) is a Mecha Architecture implementation for Java applications,
providing a unified API layer that decouples applications from infrastructure, enabling true cloud-native portability.

Cross-Cloud Deployment

One codebase, deploy to multiple cloud platforms like Alibaba Cloud, AWS, Tencent Cloud without modifying business logic.

Mecha Architecture

Based on Service Mesh concepts,下沉 infrastructure capabilities to Sidecar/SDK, letting applications focus only on business logic.

Standard APIs

Unified standard API layer supporting RPC, Configuration, Pub/Sub, State Management and more distributed capabilities.

Clean API Design

// Build RPC client
CapaRpcClient client = new CapaRpcClientBuilder().build();

// Invoke remote service
Mono result = client.invokeMethod(
    "service-app-id",
    "hello",
    "request",
    HttpExtension.POST,
    null,
    TypeRef.STRING
);

// Get result
String response = result.block();
