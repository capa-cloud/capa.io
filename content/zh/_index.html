为什么选择 Capa？

Capa (Cloud Application API) 是面向 Java 应用的 Mecha 架构实现，
提供统一的 API 层，让应用与基础设施解耦，实现真正的云原生可移植性。

{{% /blocks/lead %}} {{< blocks/section color="white" >}} {{% blocks/feature icon="fa-cloud" title="跨云部署" %}} 一套代码，同时部署到阿里云、AWS、腾讯云等多个云平台，无需修改业务代码。 {{% /blocks/feature %}} {{% blocks/feature icon="fa-layer-group" title="Mecha 架构" %}} 基于 Service Mesh 理念，将基础设施能力下沉到 Sidecar/SDK，应用只需关注业务逻辑。 {{% /blocks/feature %}} {{% blocks/feature icon="fa-code" title="标准 API" %} 统一的标准 API 层，支持 RPC、Configuration、Pub/Sub、State 等多种分布式能力。 {{% /blocks/feature %}} {{< /blocks/section >}} {{< blocks/section color="dark" >}}

简洁的 API 设计

// 构建 RPC 客户端
CapaRpcClient client = new CapaRpcClientBuilder().build();

// 调用远程服务
Mono result = client.invokeMethod(
    "service-app-id",
    "hello",
    "request",
    HttpExtension.POST,
    null,
    TypeRef.STRING
);

// 获取结果
String response = result.block();

丰富的功能支持

  • 服务调用 (RPC)
  • 配置中心 (Configuration)
  • 消息发布订阅 (Pub/Sub)
  • 状态管理 (State)
  • 可观测性 (Telemetry)
  • 数据库访问 (SQL) - Alpha
