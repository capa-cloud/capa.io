Why Capa?

Capa (Cloud Application API) is a Mecha Architecture implementation for Java applications,
providing a unified API layer that decouples applications from infrastructure, enabling true cloud-native portability.

{{% /blocks/lead %}} {{< blocks/section color="white" >}} {{% blocks/feature icon="fa-cloud" title="Cross-Cloud Deployment" %}} One codebase, deploy to multiple cloud platforms like Alibaba Cloud, AWS, Tencent Cloud without modifying business logic. {{% /blocks/feature %}} {{% blocks/feature icon="fa-layer-group" title="Mecha Architecture" %}} Based on Service Mesh concepts,下沉 infrastructure capabilities to Sidecar/SDK, letting applications focus only on business logic. {{% /blocks/feature %}} {{% blocks/feature icon="fa-code" title="Standard APIs" %}} Unified standard API layer supporting RPC, Configuration, Pub/Sub, State Management and more distributed capabilities. {{% /blocks/feature %} {{< /blocks/section >}} {{< blocks/section color="dark" >}}

Clean API Design

// Build RPC client
CapaRpcClient client = new CapaRpcClientBuilder().build();

// Invoke remote service
Mono result = client.invokeMethod(
    "service-app-id",
    "hello",
    "request",
    HttpExtension.POST,
    null,
    TypeRef.STRING
);

// Get result
String response = result.block();
